## Planning Before Implementation

Always create a plan before starting any implementation. Write the plan as a Markdown file and save it in the `plan/` directory. The content of each plan file must be written in Japanese.

## Plan Naming Convention

When creating a plan, name the file using the following format:

```
T{NNNNN}_{task-name}.md
```

- `T` prefix followed by a zero-padded 5-digit number (e.g., `T00001`, `T00042`)
- Followed by an underscore and a short, descriptive kebab-case name summarizing the task
- Example: `T00001_setup-database.md`, `T00023_add-auth-middleware.md`
